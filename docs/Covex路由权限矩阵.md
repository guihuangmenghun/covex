# Covex 路由权限矩阵

> **版本**：v3.0（基于保险公司完整组织架构重写）
> **日期**：2026-07-06
> **审计范围**：42 条路由 + 12 个菜单组 + 规则配置中心
> **角色模板**：13 个内置角色（管理员可基于模板新增自定义角色）
> **设计原则**：一个保险公司至少需要覆盖产品→精算→销售→核保→保全→理赔→财务→合规全链路

---

## 一、组织架构与角色层级

### 1.1 真实保险公司组织架构

```
                        ┌──────────────┐
                        │   总经理室    │
                        └──────┬───────┘
                               │
          ┌────────────────────┼────────────────────┐
          │                    │                    │
    ┌─────┴─────┐       ┌─────┴─────┐       ┌─────┴─────┐
    │  前 台     │       │  中 台     │       │  后 台     │
    │ (销售服务) │       │ (风控审核) │       │ (支撑管理) │
    └─────┬─────┘       └─────┬─────┘       └─────┬─────┘
          │                    │                    │
   ┌──────┼──────┐     ┌──────┼──────┐     ┌──────┼──────┐
   │      │      │     │      │      │     │      │      │
  渠道  客服  录入  核保  核赔  精算  产品  合规  财务
  销售  专员  员    师    师    师    经理  法务  结算
```

### 1.2 角色分组（4 大类 13 个角色）

| 分组 | 角色编码 | 角色名称 | 定位 | 核心职责 |
|---|---|---|---|---|
| **系统管理组** | `admin` | 管理员 | 最高权限 | 全部功能 + 系统配置 + 创建次级管理员 |
| | `sub_admin` | 次级管理员 | 运营管理 | 全部业务 + 渠道 + 佣金（无系统配置） |
| **产品精算组** | `product_mgr` | 产品经理 | 产品设计 | 产品创建/编辑/发布 + 产品规则配置 |
| | `actuary` | 精算师 | 费率定价 | 费率表管理 + 预定利率审核 + 产品定价 |
| **销售服务组** | `channel_mgr` | 渠道经理 | 渠道运营 | 渠道商管理 + 产品授权 + 佣金设置 |
| | `agent` | 录入员（代理人） | 一线操作 | 客户录入 + 投保单录入 + 保费试算 |
| | `service_rep` | 客服专员 | 客户服务 | 客户咨询 + 保全代办 + 理赔报案 |
| **核保核赔组** | `underwriter` | 核保老师 | 核保专家 | 核保工作台 + 核保规则配置 |
| | `conservation` | 保全老师 | 保全专家 | 保单服务 + 保全规则配置 |
| | `claim_handler` | 理赔老师 | 理赔专家 | 理赔工作台 + 理赔规则配置 |
| | `investigator` | 调查员 | 案件调查 | 理赔调查 + 反欺诈 + 调查报告 |
| **后台支撑组** | `finance` | 财务结算 | 资金管理 | 收付费确认 + 佣金结算 + 财务报表 |
| | `compliance` | 合规法务 | 合规审核 | 产品合规审核 + 条款审查 + 监管报送 |

### 1.3 权限层级

```
                    ┌──────────┐
                    │  管理员   │  全部权限 + 创建次级管理员
                    │  admin   │
                    └────┬─────┘
                         │
                 ┌───────┴───────┐
                 │  次级管理员     │  全部业务 + 渠道 + 佣金
                 │  sub_admin    │
                 └───────┬───────┘
                         │
     ┌───────────────────┼───────────────────┐
     │                   │                   │
 ┌───┴────┐        ┌────┴────┐        ┌─────┴────┐
 │产品精算组│        │核保核赔组│        │ 后台支撑组│
 │产品/精算│        │核保/保全/│        │ 财务/合规 │
 └───┬────┘        │理赔/调查 │        └─────┬────┘
     │             └────┬────┘              │
 ┌───┴────┐        ┌────┴────┐              │
 │销售服务组│        │ 同级互看 │              │
 │渠道/录入/│        │ (只读)  │              │
 │ 客服    │        └─────────┘              │
 └────────┘                                  │
                                             │
 权限：admin > sub_admin > 各组组长 > 各组成员 > 录入员/客服
```

**同层角色**（读写隔离，只读互通）：
- 产品经理 = 精算师（产品域内互看只读）
- 核保老师 = 保全老师 = 理赔老师 = 调查员（业务域互看只读）
- 财务 = 合规（后台域互看只读）
- 录入员 < 客服专员（客服可代录入员操作保全/理赔报案）

## 七、数据权限维度

| 角色 | 默认数据范围 | 说明 |
|---|---|---|
| admin | 全租户 | 可看到所有数据 |
| sub_admin | 全租户 | 可看到所有业务数据（系统管理除外） |
| product_mgr | 自己创建的产品 | 可看所有产品（只读），但只能编辑自己创建的 |
| actuary | 全租户（产品+费率） | 可看所有费率表，只读产品 |
| channel_mgr | 自己管理的渠道 | 只能操作自己负责的渠道商 |
| agent | 仅自己 | 只能看到自己录入的客户和投保单 |
| service_rep | 自己+分配的客户 | 可代客户操作保全/理赔 |
| underwriter | 按区域/渠道分配 | 核保案件按分配范围过滤 |
| conservation | 按区域/渠道分配 | 保全案件按分配范围过滤 |
| claim_handler | 按区域/渠道分配 | 理赔案件按分配范围过滤 |
| investigator | 被指派的案件 | 只看到分配给自己的调查任务 |
| finance | 全租户（财务数据） | 收付费、佣金、报表 |
| compliance | 全租户（只读+审核） | 所有产品/条款可看，只能做合规审核 |

## 八、同级互看规则

```
核保老师A 可以看到：
  ├── 自己的核保案件（读写）
  ├── 保全老师B 的保全案件（只读）
  ├── 理赔老师C 的理赔案件（只读）
  └── 调查员D 的调查报告（只读）

产品经理E 可以看到：
  ├── 自己创建的产品（读写）
  ├── 精算师F 的费率表（只读）
  └── 其他产品经理的产品（只读）

录入员G 可以看到：
  ├── 自己录入的客户和投保单（读写）
  ├── 不能看到其他录入员的数据
  └── 可查看自己投保单的核保进度（只读）

客服专员H 可以看到：
  ├── 分配给自己的客户（读写）
  ├── 可代办保全/理赔报案
  └── 可查看客户的保单和投保单（只读）
```

## 十、为什么需要这些角色（保险公司实际业务场景）

| 角色 | 为什么需要 | 不配此角色的风险 |
|---|---|---|
| 精算师 | 费率定价是保险核心，必须由专人管控 | 费率错误→监管处罚→公司亏损 |
| 合规法务 | 银保监会要求保险公司有合规岗位 | 条款违规→罚款→停业整顿 |
| 调查员 | 保险欺诈年损失数百亿，需专人调查 | 欺诈漏赔→赔付率失控 |
| 财务结算 | 收付费、佣金结算涉及资金安全 | 资金错乱→审计问题→法律风险 |
| 客服专员 | 保全/理赔报案量大，需专人处理 | 客户投诉→监管转办→声誉损失 |
| 产品经理 | 产品生命周期管理需要专人负责 | 产品配置混乱→销售错误 |
