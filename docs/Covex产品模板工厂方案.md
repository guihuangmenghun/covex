# Covex 产品库 & 模板工厂方案

> **版本**：v1.1
> **日期**：2026-07-06
> **目标用户**：产品经理（不需要保险专业知识）
> **核心理念**：产品库 + 模板工厂。选模板 → 填参数 → 出产品。保险知识由系统预置，PM 只需做选择题。

---

## 〇、产品库三层架构

产品经理创建产品时，有三个来源可选：

```
┌─────────────────────────────────────────────────────────┐
│                    产品库（三层结构）                      │
│                                                         │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 第 1 层：系统模板（平台预置，不可删除）            │    │
│  │ 定期寿险 / 终身寿险 / 重疾险 / 车险 / 财产险 / 意外险│    │
│  │ → 任何保险公司都能用，开箱即用                     │    │
│  └─────────────────────────────────────────────────┘    │
│                         │                               │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 第 2 层：公司模板（管理员/精算师自建）              │    │
│  │ 如：“XX人寿专属定寿模板”、“银保渠道年金模板”       │    │
│  │ → 基于系统模板克隆，调整为公司专属参数             │    │
│  └─────────────────────────────────────────────────┘    │
│                         │                               │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 第 3 层：已发布产品（可直接克隆为新产品）           │    │
│  │ 如：“XX定寿v2.0” → 克隆 → “XX定寿v3.0”           │    │
│  │ → 老产品升级最快的方式                             │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

| 层级 | 来源 | 谁创建 | 谁使用 | 可删除？ |
|---|---|---|---|---|
| 系统模板 | 平台预置 | 系统内置 | 所有租户 | 不可删除，可停用 |
| 公司模板 | 基于系统模板克隆 | 管理员/精算师 | 本租户 | 可停用 |
| 已发布产品 | 基于模板或空白创建 | 产品经理 | 本租户 | 下架（不删除） |

**产品经理的 3 种创建路径**：

| 场景 | 选择 | 操作量 |
|---|---|---|
| 常规险种，第一次做 | 系统模板 → 填参数 | 最少（8-12 个数字） |
| 公司已有类似产品模板 | 公司模板 → 微调参数 | 少（3-5 个调整） |
| 老产品升级新版本 | 克隆已发布产品 → 改参数 | 最少（改差异部分） |

## 一、问题分析

### 1.1 产品经理的痛点

创建一个保险产品需要配置：
- 产品主表（product_type、capabilities、attributes 等 20+ 字段）
- 保障定义（3-8 个 coverage，每个有 10+ JSON 属性）
- 缴费规则（1-3 个 premium plan）
- 规则引用（核保/校验/费率计算等 5-10 条规则）
- 费率表（维度定义 + 几十到几百行费率数据）

**不懂保险的人根本不知道填什么。**

### 1.2 解决方案

```
┌─────────────────────────────────────────────────────┐
│              产品模板工厂                             │
│                                                     │
│  Step 1: 选模板（6 个预置模板，覆盖主流险种）         │
│  Step 2: 填参数（8-12 个关键数字，全部有默认值）      │
│  Step 3: 确认生成（系统自动创建完整产品配置）         │
│                                                     │
│  结果：一个完整可用的保险产品（含保障/缴费/规则/费率） │
└─────────────────────────────────────────────────────┘
```

## 二、数据模型

### 2.1 新增表：`ins_product_template` — 产品模板

| 字段名 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT AUTO_INCREMENT | 主键 |
| `tenant_id` | BIGINT | 租户ID |
| `template_code` | VARCHAR(30) | 模板编码，如 `TERM_LIFE`、`WHOLE_LIFE` |
| `template_name` | VARCHAR(100) | 模板名称，如 "定期寿险模板" |
| `template_desc` | VARCHAR(500) | 模板说明（给 PM 看的，解释这个模板适合什么场景） |
| `product_type` | TINYINT | 对应的产品分类（1-寿险 2-意外险...） |
| `icon` | VARCHAR(50) | 前端展示图标 |
| `sort_order` | INT | 排序 |
| `is_active` | TINYINT(1) | 是否启用 |
| `template_data` | JSON | **完整模板数据**（核心） |
| `param_schema` | JSON | **PM 需要填的参数定义**（前端渲染表单用） |
| `created_by` | VARCHAR(50) | 创建人 |
| `updated_by` | VARCHAR(50) | 修改人 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 修改时间 |
| `is_deleted` | TINYINT(1) | 软删除 |
| `deleted_at` | DATETIME | 删除时间 |

> **唯一索引**：`UNIQUE(tenant_id, template_code)`

**模板来源分类**：

| `tenant_id` | `template_code` 前缀 | 说明 |
|---|---|---|
| `0` | `SYS_` | 系统模板（平台预置，所有租户可见） |
| `>0` | `COMPANY_` | 公司模板（本租户专属） |

### 2.1.1 `ins_product` 表新增字段

| 字段名 | 类型 | 说明 |
|---|---|---|
| `template_source` | TINYINT | 产品来源：1-空白创建，2-系统模板，3-公司模板，4-克隆产品 |
| `template_ref_id` | BIGINT | 来源模板ID或来源产品ID（空白创建时为NULL） |

> 这两个字段加到现有 `ins_product` 表，用于追溯产品是从哪个模板/产品创建的。

## 三、预置模板清单（6 个）

### 3.1 定期寿险模板 `TERM_LIFE`

| 项目 | 预置值 |
|---|---|
| 适合场景 | 家庭经济支柱的身故保障，保到 60/70 岁 |
| 保障责任 | 身故保险金（必选）、全残保险金（必选） |
| 缴费方式 | 年交，5/10/15/20/30 年可选 |
| 核保规则 | 年龄校验 + 健康告知 + 职业类别 + 累计保额 |
| 费率维度 | 年龄 × 性别 × 保障期间 × 缴费期间 |
| PM 需填 | 产品名称/编码、投保年龄范围、保额范围、保障期间选项 |

### 3.2 终身寿险模板 `WHOLE_LIFE`

| 项目 | 预置值 |
|---|---|
| 适合场景 | 终身保障 + 现金价值（有储蓄功能） |
| 保障责任 | 身故保险金（必选）、全残保险金（必选）、保单借款（能力） |
| 缴费方式 | 年交，10/20/30 年或趸交 |
| 核保规则 | 年龄校验 + 健康告知 + 职业类别 + 财务核保 |
| 费率维度 | 年龄 × 性别（终身） |
| PM 需填 | 产品名称/编码、投保年龄范围、保额范围 |

### 3.3 重疾险模板 `CRITICAL_ILLNESS`

| 项目 | 预置值 |
|---|---|
| 适合场景 | 确诊即赔的重大疾病保障 |
| 保障责任 | 重疾保险金（必选）、中症保险金（可选）、轻症保险金（可选）、豁免保费（可选） |
| 缴费方式 | 年交，10/20/30 年 |
| 核保规则 | 年龄校验 + 健康告知（严格） + 职业类别 + 等待期 90/180 天 |
| 费率维度 | 年龄 × 性别 × 保障期间 × 缴费期间 |
| PM 需填 | 产品名称/编码、病种数量、等待期、保额范围、是否含中症/轻症 |

### 3.4 车险模板 `AUTO_INSURANCE`

| 项目 | 预置值 |
|---|---|
| 适合场景 | 机动车辆保险（交强险 + 商业险） |
| 保障责任 | 交强险（必选）、车损险（可选）、三者险（可选）、车上人员（可选） |
| 缴费方式 | 趸交（一年一交） |
| 核保规则 | 车辆年限 + 历史赔付（NCD） + 车型校验 |
| 费率维度 | 车型 × 车龄 × NCD系数 × 地区系数 |
| PM 需填 | 产品名称/编码、NCD 折扣上限、三者险保额选项 |

### 3.5 财产险模板 `PROPERTY_INSURANCE`

| 项目 | 预置值 |
|---|---|
| 适合场景 | 企业/家庭财产保障 |
| 保障责任 | 火灾爆炸（必选）、暴雨洪水（可选）、盗窃（可选）、水暖管爆裂（可选） |
| 缴费方式 | 趸交（一年一交） |
| 核保规则 | 财产类型 + 标的估值 + 地区风险 |
| 费率维度 | 财产类型 × 灾因 × 地区 × 保额 |
| PM 需填 | 产品名称/编码、适用财产类型、免赔额/率、保额上限 |

### 3.6 意外险模板 `ACCIDENT_INSURANCE`

| 项目 | 预置值 |
|---|---|
| 适合场景 | 意外伤害保障（个人/团体） |
| 保障责任 | 意外身故（必选）、意外伤残（必选）、意外医疗（可选）、意外住院津贴（可选） |
| 缴费方式 | 趸交（一年一交） |
| 核保规则 | 年龄校验 + 职业类别（1-4 类可保） |
| 费率维度 | 年龄 × 职业类别 |
| PM 需填 | 产品名称/编码、保额范围、职业类别限制、是否含意外医疗 |

```
GET    /api/product-template                    # 查询模板列表（仅 active）
```

```java
public Product createFromTemplate(String templateCode, Map<String, Object> params) {
    // 1. 加载模板
    ProductTemplate template = templateMapper.selectByCode(templateCode);
    
    // 2. 解析 template_data
    TemplateData data = JSON.parseObject(template.getTemplateData(), TemplateData.class);
    
    // 3. 用 params 覆盖模板中的可配置参数
    data.getProduct().getAttributes().put("max_insured_age", params.get("max_insured_age"));
    data.getProduct().getAttributes().put("max_maturity_age", params.get("max_maturity_age"));
    // ... 其他参数替换
    
    // 4. 创建产品主表
    Product product = new Product();
    product.setProductCode(params.get("product_code"));
    product.setProductName(params.get("product_name"));
    product.setProductType(template.getProductType());
    // ... 从模板复制
    productMapper.insert(product);
    
    // 5. 批量创建保障定义
    for (CoverageTemplate cov : data.getCoverages()) {
        ProductCoverage coverage = new ProductCoverage();
        coverage.setProductId(product.getId());
        // ... 从模板复制
        coverageMapper.insert(coverage);
    }
    
    // 6. 创建缴费计划
    // 7. 创建 coverage-premium 关联
    // 8. 创建规则引用
    // 9. 创建空费率表（待精算师填写）
    
    // 10. 记录 changelog
    return product;
}
```

## 六、模板与角色的关系

```
产品经理的工作流：
                                                    
  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
  │ 选模板    │ →  │ 填参数    │ →  │ 生成草稿  │ →  │ 提交审批  │
  │ (选择题)  │    │ (填空题)  │    │ (自动)    │    │          │
  └──────────┘    └──────────┘    └──────────┘    └────┬─────┘
                                                       │
                              ┌─────────────────────────┤
                              │                         │
                        ┌─────▼─────┐            ┌─────▼─────┐
                        │ 精算师     │            │ 合规法务   │
                        │ 填费率表   │            │ 审核条款   │
                        └─────┬─────┘            └─────┬─────┘
                              │                         │
                              └────────────┬────────────┘
                                           │
                                     ┌─────▼─────┐
                                     │ 管理员审批  │
                                     │ 发布上架    │
                                     └───────────┘
```

| 角色 | 在产品创建中的职责 | 需要保险知识？ |
|---|---|---|
| 产品经理 | 选模板 + 填参数 → 生成产品草稿 | 不需要（模板已预置） |
| 精算师 | 填写费率表的具体费率值 | 需要（精算专业） |
| 合规法务 | 审核产品条款合规性 | 需要（法律专业） |
| 管理员 | 最终审批发布 | 不需要（流程审批） |
