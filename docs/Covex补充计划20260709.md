## P0 — 遗留 Bug 修复（阻塞业务正确性）

### Task 0.1 健康档案 JSON 结构修复 [Bug]
- **问题**：`PUT /api/customer/{id}/health` 返回 500，前端发送 `{ medicalHistory: [], familyHistory: [], currentMedications: [] }` 嵌套对象，后端 `@RequestBody CustomerInsuredEntity` 无法反序列化
- **错误日志**：`JSON parse error: Cannot deserialize value of type LinkedHashMap from Array value (token JsonToken.START_ARRAY)`
- **根因**：前端 `healthForm` 结构与后端 `CustomerInsuredEntity` 字段定义不匹配
- **预估**：1 小时
- **文件**：
  - 后端：`covex-service/.../entity/CustomerInsuredEntity.java`、`covex-service/.../service/CustomerService.java`、`covex-web/.../controller/CustomerController.java`
  - 前端：`Covex-ui/src/views/customer/CustomerDetail.vue`
- **方案**：
  - 方案 A（推荐）：后端新增 `HealthRecordDTO`，包含 `medicalHistory`/`familyHistory`/`currentMedications` 三个 JSON 数组字段，Controller 接收 DTO 后写入 `CustomerInsuredEntity` 的 JSON 字段
  - 方案 B：前端改造 `healthForm` 结构匹配后端实体字段
- **验证**：
- [ ] 确认 `CustomerInsuredEntity` 中健康档案相关字段定义（JSON 类型 or 独立字段）
- [ ] 选择方案并实现（推荐方案 A：新增 DTO）
- [ ] 前端填写健康档案 → 保存成功 → 刷新页面数据回显正确
- [ ] 理赔流程中核保健康校验（UwHealthComponent）仍能正常读取健康数据
