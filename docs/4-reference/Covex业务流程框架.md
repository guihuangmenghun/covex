# Covex 业务流程框架

> 本文档定义 Covex 平台的完整业务生命周期，作为运营域数据模型设计的依据。

---

## 一、业务全景（6 个阶段）

```
产品设计 → 渠道销售 → 承保出单 → 保单服务 → 理赔处理 → 保单终止
   ↑                                                          |
   └──────────────── 产品迭代（基于理赔/运营数据反馈）─────────┘
```

---

## 二、阶段一：产品设计与上架

### 流程

```
产品经理创建产品 → 配置保障责任 → 配置缴费计划 → 关联费率表
    → 关联规则链(LiteFlow) → 提交审批 → 精算/合规审核 → 发布上架
```

### 参与角色
- **产品经理**：创建产品、配置属性、上传条款文档
- **精算师**：审核费率表、预定利率
- **合规人员**：审核条款合规性
- **系统管理员**：审批流程管理

### 涉及数据实体（产品配置域 — 已完成）
- ins_product / ins_product_coverage / ins_product_premium
- ins_product_rule / ins_product_document / ins_rate_table
- ins_product_changelog

### 待定义实体
- **审批记录表**：谁审批、审批意见、审批时间（或复用 ins_product_changelog）

## 三、阶段二：渠道与销售

### 3.1 渠道商管理

```
渠道商入驻 → 资质审核 → 签约 → 授权产品 → 分配账号 → 日常运营
```

**渠道类型**：
| 编码 | 类型 | 说明 |
|---|---|---|
| 1 | 个人代理人 | 持证保险代理人 |
| 2 | 公司直销 | 保险公司自有销售团队 |
| 3 | 专业代理公司 | 保险代理公司 |
| 4 | 银行/邮政 | 银行保险渠道 |
| 5 | 经纪公司 | 保险经纪公司 |
| 6 | 互联网平台 | 线上销售平台 |

### 3.2 渠道商授权

```
渠道商 → 绑定可售产品列表 → 设置佣金比例 → 分配销售区域
```

### 参与角色
- **渠道管理员**：渠道商入驻审核、产品授权、佣金设置
- **渠道商**：查看可售产品、发起投保

### 待定义实体
- **渠道商表**（ins_channel）：基本信息、资质、状态
- **渠道商-产品授权表**（ins_channel_product）：可售产品、佣金比例、销售区域
- **渠道商账号表**（ins_channel_user）：登录账号、权限

## 四、阶段三：投保与承保

### 4.1 投保流程

```
投保人选择产品 → 填写投保信息（投保人/被保人/受益人）
    → 选择保障责任和缴费计划 → 健康告知（如需要）
    → 系统校验（LiteFlow validate 链）
    → 核保（LiteFlow underwrite 链）
        → 自动核保通过 → 进入计费
        → 需人工核保 → 转核保员审核 → 核保结论（标准体/加费/除外/拒保/延期）
    → 保费计算（Aviator 表达式 + Redis 费率表）
    → 生成投保单 → 等待支付
```

### 4.2 支付与出单

```
投保人支付（对接支付通道）
    → 支付回调 → 支付确认
    → 生成保单号 → 保单生效
    → 异步触发：生成电子保单、发送通知、佣金计算、再保安排
```

### 4.3 核保流程（详细）

```
核保规则执行（LiteFlow 链）
    ├── 年龄校验
    ├── 健康告知校验
    ├── 职业类别校验
    ├── 保额上限校验
    ├── 累计风险保额校验
    └── 财务核保（高保额）

核保结论：
    1 → 标准体通过（正常费率）
    2 → 次标准体加费（增加保费）
    3 → 除外责任（部分不保）
    4 → 延期（观察期后再议）
    5 → 拒保（不接受投保）
    6 → 转人工（复杂案件）
```

### 参与角色
- **投保人**：填写投保信息、支付保费
- **被保人**：保险标的（人）
- **受益人**：保险金领取人
- **渠道商/代理人**：协助投保、代收保费
- **核保员**：人工核保审核
- **系统**：自动校验、自动核保、保费计算

### 待定义实体
- **客户表**（ins_customer）：投保人/被保人/受益人基本信息
- **投保人扩展表**（ins_customer_applicant）：投保人特有属性（收入/婚姻/社保）
- **被保人扩展表**（ins_customer_insured）：被保人特有属性 + 健康档案
- **银行账户表**（ins_customer_bank_account）：缴费扣款/理赔收款账户
- **联系地址表**（ins_customer_address）：户籍/常住/工作/通讯地址
- **投保单表**（ins_proposal）：投保申请信息、状态流转
- **保单主表**（ins_policy）：保单基本信息、状态、生效/到期日
- **保单险种明细表**（ins_policy_coverage）：保单关联的产品+责任+保额+保费
- **保单缴费计划表**（ins_policy_premium）：实际缴费安排
- **核保记录表**（ins_underwriting_record）：核保过程、结论、人工审核意见
- **支付记录表**（ins_payment）：支付流水、状态
- **佣金记录表**（ins_commission）：渠道佣金计算结果

## 五、阶段四：保单服务（保全/续期）

### 5.1 续期收费

```
续期保费到期 → 系统生成续期账单
    → 发送缴费提醒（短信/推送）
    → 投保人缴费 → 支付确认 → 保单续期
    → 超宽限期未缴 → 保单中止 → 2年内可申请复效
    → 超2年未复效 → 保单终止
```

### 5.2 保全变更

```
保单持有人申请保全
    ├── 信息变更（联系方式、地址、受益人变更）→ 直接生效
    ├── 保额变更（加保/减保）→ 需核保 → 补退保费 → 生效
    ├── 缴费方式变更 → 生效
    ├── 保单借款 → 计算利息 → 放款
    ├── 保单复效 → 需核保 → 补缴保费 → 生效
    └── 退保 → 计算退保金 → 支付 → 保单终止
```

### 5.3 犹豫期退保

```
保单签收后 N 天内（通常 10-15 天）
    → 申请犹豫期退保 → 全额退还保费（扣工本费）→ 保单终止
```

### 参与角色
- **保单持有人**：发起保全申请
- **保全人员**：审核保全申请
- **系统**：续期账单生成、缴费提醒、退保金计算

### 待定义实体
- **续期账单表**（ins_renewal_bill）：续期保费、到期日、缴费状态
- **保全申请表**（ins_endorsement）：保全类型、申请信息、审批状态
- **保全变更记录表**（ins_endorsement_change）：变更前后的值
- **保单借款表**（ins_policy_loan）：借款金额、利率、还款记录

## 六、阶段五：理赔处理

### 6.1 理赔流程

```
出险报案 → 提交理赔材料（医疗单据/事故证明/死亡证明等）
    → 立案 → 理赔审核（LiteFlow claim 链）
        ├── 保单有效性校验
        ├── 保障责任匹配
        ├── 等待期校验
        ├── 免赔额计算
        ├── 赔付限额校验
        └── 反欺诈规则
    → 理赔结论
        ├── 正常赔付 → 计算赔付金额 → 支付 → 结案
        ├── 部分赔付 → 计算赔付金额 → 支付 → 结案
        ├── 拒赔 → 通知 → 结案
        └── 需调查 → 转调查员 → 补充材料 → 重新审核
```

### 6.2 赔付计算

```
赔付金额 = f(保额, 免赔额, 赔付比例, 累计已赔付, 限额)

示例（医疗险）：
  claim = (medical_cost - deductible) * claim_ratio
  claim = min(claim, per_accident_limit - already_paid)

示例（重疾险）：
  claim = sum_insured * claim_ratio  // 通常 100%
```

### 参与角色
- **报案人**：出险报案（可以是被保人、投保人或代理人）
- **理赔员**：审核理赔材料、做出理赔结论
- **调查员**：复杂案件调查
- **系统**：自动校验、赔付计算、反欺诈规则

### 待定义实体
- **理赔案件表**（ins_claim）：案件基本信息、状态
- **理赔材料表**（ins_claim_document）：上传的理赔材料
- **理赔审核记录表**（ins_claim_review）：审核过程、结论
- **赔付记录表**（ins_claim_payment）：赔付金额、支付状态

## 七、阶段六：保单终止

### 终止原因

| 编码 | 原因 | 说明 |
|---|---|---|
| 1 | 满期终止 | 保险期间届满 |
| 2 | 退保终止 | 保单持有人主动退保 |
| 3 | 犹豫期退保 | 犹豫期内退保 |
| 4 | 理赔终止 | 全部保额赔付完毕 |
| 5 | 身故终止 | 被保人身故 |
| 6 | 复效超期 | 中止超 2 年未复效 |
| 7 | 拒保终止 | 核保拒绝 |

### 终止后处理

```
保单终止 → 计算退保金/满期金（如有）→ 支付 → 释放风险保额 → 归档
```

## 十一、设计注释

### 1. 产品配置域与运营域的关系
- 产品配置域（11 张表）定义"产品模板"——产品可以怎么卖
- 运营域（29 张表）记录"业务实例"——谁买了什么、付了多少、赔了多少
- 投保时从产品配置域"实例化"到保单：product → policy, coverage → policy_coverage, premium → policy_premium

### 2. LiteFlow 链在各阶段的应用
- 投保链：validate → underwrite → calculate → issue → notify
- 保全链：validate → calculate → approve → execute → notify
- 理赔链：validate → match_coverage → calculate → review → pay → notify
- 续期链：generate_bill → remind → collect → confirm → renew

### 3. 后续可扩展的业务流程
- 再保安排（分出/分入）
- 团险批量投保
- 互联网保险（对接第三方平台）
- 监管报送（保监会/银保监会数据上报）
