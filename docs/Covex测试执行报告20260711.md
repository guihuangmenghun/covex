## 根因分析

### 第三轮发现的 500 错误根因（第四轮已解决）

第三轮测试发现的 500 错误分为两类：

**假阳性（测试 Agent 未读取 OpenAPI 规范）：**
- `POST /api/customer` — 测试未传必填字段 `idNo`，补传后正常
- `PUT /api/underwriting/{id}/approve` — 测试使用了错误路径，正确路径为 `POST /api/underwriting/manual/{id}`
- `POST /api/payment` — 测试使用了错误路径，正确路径为 `POST /api/payment/create`
- `POST /api/product-template/create-product` — 测试使用了 `templateId`，正确字段为 `templateCode`
- `POST /api/user/{id}/roles` — 测试请求体格式错误，正确格式为 JSON 数组 `[7]`

**真实 Bug（已编码修复）：**
- `GET /api/product?size>=50` — V24 清理 sale_channel 脏数据后修复
- `GET /api/customer/{id}` 无 healthArchive — 新增 `GET /api/customer/{id}/health` 端点
- ProposalDetail 产品性质显示数字 — 新增 `productNatureLabel()` 使用 dictStore
- Product capabilities 缺少 shared_amount — V25 SQL 为所有产品补充
